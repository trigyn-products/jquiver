# Naming Conventions

## Purpose
Document naming conventions for JQuiver metadata and custom schemas.

## When to use this file
Use this file before naming routes, forms, grids, APIs, datasources, file bins, and resource keys.

## Related files
- `coding-standards.md`
- `metadata-table-reference.md`

## Known facts
- Observed platform tables use `jq_` prefixes.
- Observed ARK domain tables use `ark_` prefixes in the main schema.
- Observed SBI interview tables use `intrv_` prefixes.
- Observed route names may be short aliases such as `jdd`, `jdo`, `appdt`, and `intrv`.
